## 0. 项目目标（一句话）
做一个多语种（中/日/韩/英/泰）社区风控 Agent：识别黑评/引战/辱骂/广告引流 → 处置（隐藏/删除/警告/禁言/封禁）→ 三振自动拉黑 → 可解释、可复核、可统计。

## 1. 必做功能（MVP 不可删）
- 内容接入：发帖/评论/回复
- 多语种识别：zh/ja/ko/en/th
- 风控判定：规则 + 模型（先规则跑通，再接火山引擎）
- 处置动作（Demo 平台必须能真的执行）：隐藏/删除/警告/禁言/封禁
- 三振机制：同用户违规累计 3 次自动封禁（可配置时间窗口）
- 善意评判：降低误杀；不确定进入复核队列
- 审计日志：保存证据片段、规则命中、模型输出摘要、最终决策原因
- 后台管理页：列表、搜索、复核改判、解封、策略开关
- 数据分析：广告治理指标 + 留存/回访 + 风控效果（命中率/误杀率/处理时延）

## 2. 准备清单（开工前）
- 本机：Python 3.11+、VSCode
- 数据库：SQLite（当前阶段），后续可切 Oracle 19
- 火山引擎：你已有 API Key（阶段 4 再接入）
- 密钥管理：只放 `.env`（本机），不要提交

## 3. 执行顺序（必须按这个来）
### 阶段 1：骨架可运行
- 创建后端服务（FastAPI）+ 健康检查接口
- 本地启动成功

### 阶段 2：数据可落库
- 建表：用户、内容、风控结果、处罚记录、复核记录、指标事件
- 发布评论 → 入库 → 返回内容 ID

### 阶段 3：风控可判定（规则版）
- 规则命中：脏话、威胁、挂人、联系方式/引流
- 策略引擎：输出动作（隐藏/警告/禁言/封禁）
- 三振：累计到 3 次自动封禁

### 阶段 4：接入火山模型（多语种语义）
- 模型结构化 JSON 输出：类别、置信度、证据片段、目标对象
- 善意评判 Gate：批评≠攻击；引用/反讽/不确定走复核

### 阶段 5：管理后台（好看 + 可用）
- 内容列表、命中证据、复核改判、封禁/解封、策略配置
- 交互重点：一眼看到“为什么被判”

### 阶段 6：数据分析
- 广告治理：命中率/漏网率/处置时延/复发率
- 留存：D1/D7 回访；误杀关联（改判率/申诉率）
- 按“策略版本”做对比报表

## 4. 卡住怎么办（固定排查流程）
- 先看服务能不能启动（起不来先别碰数据库/模型）
- 再看能不能入库（连接、建表、权限）
- 再看能不能判定（先规则后模型；模型失败要降级）
当你卡住时，保留两样信息给我就够：
- 你运行的命令
- 完整报错文本/截图

## 5. 注意事项（必看）
- 不宣称能直接删小红书/抖音：做成“平台适配器”，无权限时输出建议处置 + 证据
- 必须有复核与解封：否则风控不可用且伤留存
- 密钥与密码只放 `.env`：不要写进代码/截图
- 策略要版本化：每次策略变更记录版本号，指标按版本对比
- 统一 UTF-8：多语种常见坑

## 6. 面试讲法（30 秒）
用户发内容 → 预处理与语言识别 → 规则筛 +（后续 LLM 结构化判定）→ 善意评判降误杀 → 策略引擎决定动作 → 记录证据与审计 → 三振自动封禁 → 运营后台复核 → 指标评估对广告治理与留存的影响。
